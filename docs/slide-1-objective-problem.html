Introduction

This slide presents the Objectives and Problem Statement of the Movie Ticket Sales System — a full-stack web application designed to modernize how customers discover movies, book seats, and pay for tickets online.

🎯 Objective

The primary goal of this project is to build a complete online cinema platform that enables customers to discover movies, browse available showtimes, reserve seats, and complete payment — all within one seamless digital experience.

The system replaces traditional manual box-office operations with an automated, role-based web application. It serves four distinct user types from a single unified platform: Customers, Cinema Staff, Cinema Managers, and System Administrators.

In terms of scale, the platform supports 4 user roles, exposes 20+ API modules, and enforces 100% role-based authentication via JWT tokens to ensure that each user only accesses the features and data permitted for their role.

⚡ Problem Statement

Problem 1 — Long queue times: Traditional box-office queues create long wait times and a poor experience for moviegoers, especially during peak hours and holiday seasons.

Problem 2 — Seat management errors: Manual seat assignment is highly error-prone, frequently resulting in double-bookings and scheduling conflicts that frustrate both customers and staff.

Problem 3 — No centralized staff tool: Cinema staff currently lack a unified system to handle ticket check-in, payment processing, and concession management simultaneously and in real time.

Problem 4 — No loyalty or promotion system: Without an integrated promotion and loyalty program, it is difficult for cinemas to retain repeat customers or increase revenue per transaction.

Problem 5 — No unified admin overview: Administrators managing multiple cinema chains struggle to oversee user accounts, operational data, and performance metrics without a single centralized dashboard.

Tech Stack

The platform is built with React on the frontend for a dynamic, responsive UI; Spring Boot on the backend with a RESTful API architecture; MySQL as the relational database; Docker for containerized deployment; and JWT for secure, stateless authentication across all roles.
